HPLC Audit Trail Monitoring Platform

분석 장비에서 생성되는 Audit Trail 데이터를 자동으로 전수 점검하는 데이터 무결성(DI) 모니터링 플랫폼. 기존 15% 샘플링 수작업 방식을 100% 자동화하여 장비당 점검 시간을 23시간에서 3시간으로 단축하고, 규제기관 지적 리스크를 사전에 차단합니다.

Skills & Technologies

Python Plotly / Dash Pandas / NumPy LangChain Ollama (nemotron-nano:8b) FAISS RAG PyPDFLoader Z-Score Anomaly Detection Rule-based Filtering xlsxwriter GMP Compliance

Problem

규제기관의 데이터 무결성(Data Integrity, DI) 평가 지침이 강화되면서, 분석 장비에서 생성되는 Audit Trail 데이터에 대한 정기 점검이 필수 요건이 되었습니다. Audit Trail은 개별 분석 스텝에서의 사용자 행위와 데이터 완전성 위반 여부를 기록하는 핵심 메타데이터로, 규정을 벗어난 행위가 발견될 경우 제조정지 등 행정처분 리스크가 존재합니다.

분석 장비 한 대에서만 연간 약 3만 건 이상의 Audit Trail이 생성됩니다. 기존에는 담당자가 수작업으로 검토해야 했으며, 현실적으로 전체의 약 15%만 샘플링 점검하는 수준에 그쳐 데이터 무결성 위험이 상시 잠재해 있었습니다. 장비 소프트웨어 자체에는 시간 갭 비교, 비정상 행위 필터링 등의 탐지 기능이 존재하지 않아 수천 건의 이력을 사람이 하나하나 확인하는 것은 사실상 불가능한 상황이었습니다.

플랫폼 탐지 시나리오

시나리오 — 재처리 기록 누락 사전 방지

아래는 본 플랫폼의 시간 갭 분석 기능이 어떤 방식으로 잠재적 데이터 무결성 위반을 사전에 방지할 수 있는지 보여주는 시나리오입니다.

Day 1
HPLC 분석 완료 및 데이터 자동 획득(Acquisition). 소프트웨어가 자동 적분을 수행하며 특정 피크를 독립 피크로 검출.
Day 4
분석자가 1차 수동적분(Manual Integration) 수행. SOP 기준상 처리가 필요한 피크가 휴먼 에러로 누락된 상태로 처리됨.
Day 11
검토자가 1차 결과 리뷰 중 미준수 사항 발견, 2차 수동적분(재처리) 수행. 그러나 Audit Trail 점검일지에는 재처리 기록 없이 단순 수동적분 여부만 기재됨.
탐지
본 플랫폼의 시간 갭 분석 기능이 Day 1(생성)과 Day 11(최종 처리) 사이 약 10일간의 시간 갭을 자동으로 감지하여 이상 케이스로 분류. 검토자가 근거·사유·기록의 적절성을 재점검하도록 안내.
→ 기록 누락이라는 잠재적 데이터 무결성 이슈를 사전에 방지.

기존 방식 vs. 플랫폼 비교

비교 항목 기존 수작업 방식 본 플랫폼

장비당 점검 시간 ✗ 약 23시간 소요 ✓ 약 3시간으로 단축
동일 샘플 재처리 감지 ✗ 불가 — 수많은 이력 중 수동 확인 필요 ✓ 자동 감지 및 알림
생성·처리 시간 갭 비교 ✗ 불가 — 날짜 비교 기능 부재 ✓ 자동 계산 및 시각화
시간 갭 기준 필터링/정렬 ✗ 불가 — 필터링 기준 부재 ✓ 이상치 구간 설정 및 sorting
Action 유형별 분석 ✗ 불가 — 개별 로그만 확인 가능 ✓ 유형별 집계·시각화·추이 분석
규제 문서 조회 ✗ 수동 검색 (문서 개별 열람) ✓ RAG 기반 AI 챗봇 즉시 조회
이상 패턴 통계 분석 ✗ 불가 ✓ Z-Score 기반 임계값 자동 탐지
리포트 내보내기 ✗ 불가 — 화면 캡처 수준 ✓ Excel 3-sheet 자동 생성 (xlsxwriter)
